# 肉鸽模式 GDD

> FF14 战斗体验 × 杀戮尖塔式爬塔肉鸽的同人小品游戏

---

## 1. 游戏概述

### 1.1 核心体验

玩家沿一条分支路径向上攀爬，途经小怪、精英怪、随机事件和奖励节点，最终挑战层boss。战斗系统复刻FF14的职业技能循环与boss机制处理，爬塔层提供每局不同的路径选择与装备构筑体验。支持断点续玩——玩家可以随时离开，下次从当前节点继续。

### 1.2 关键参数

| 参数         | 值                             |
| ------------ | ------------------------------ |
| 单轮目标时长 | 20-30 分钟                     |
| 玩家等级上限 | 15                             |
| 初始可选职业 | 3（剑术师 / 咒术师 / 弓箭手）  |
| 总职业数     | >3（通过局内武器掉落解锁更多） |
| 视角/操作    | 沿用已有 FF14 boss战模拟器     |
| 存档         | 支持断点续玩，按节点保存进度   |

## 2. 局内系统（爬塔肉鸽）

### 2.1 塔层结构

每局游戏是一座塔，塔由若干层组成，每层包含多个节点，节点之间形成分支路径（类似杀戮尖塔的地图）。

```
        [BOSS]              ← 层 Boss
       /      \\
     [?]      [E]           ← 随机事件 / 精英怪
    / | \\    / | \\
  [M] [R] [M] [M] [?]      ← 小怪 / 奖励 / 事件
   |   |   |   |   |
  [M] [?] [E] [R] [M]
   \\   |   |   |  /
    \\  |   |  | /
     [START]                ← 起点
```

玩家在每个节点完成后，从可达的下层节点中选择下一步走向。路径选择是核心策略：是稳扎稳打刷小怪攒水晶，还是冒险挑战精英怪博更好的掉落。

### 2.2 节点类型

| 节点      | 图标 | 内容                                                                              |
| --------- | ---- | --------------------------------------------------------------------------------- |
| 小怪战斗  | ⚔️   | 较弱的敌人，击杀获得水晶。低风险低回报                                            |
| 精英战斗  | 💀   | 较强的敌人，有独特机制。击杀必定掉落 3 选 1 奖励（见 2.7）                        |
| Boss 战斗 | 👑   | 每座塔的最终关卡（见 2.13）                                                       |
| 篝火      | 🔥   | 安全节点，可升级/配装/奉献；不论分支，最中间和最后一个可选层必定是篝火（见 2.12） |
| 奖励      | 🎁   | 免费获得水晶、魔晶石或其他资源                                                    |
| 随机事件  | ❓   | 随机触发特殊事件（见 2.3）                                                        |

### 2.3 随机事件

随机事件节点会触发一个随机事件，可能包括：

- 发现一个宝箱，但打开需要付出代价（消耗水晶/HP）
- 遇到一个 NPC，提供交易（用魔晶石换魔晶石、用水晶换武器、用水晶提升魔晶石的品质等）
- 触发 buff/debuff（持续若干节点）
- 遭遇陷阱（战斗前受到伤害/debuff）
- 其他惊喜或危机

> 具体事件池 TODO，核心原则：每个事件都应该包含一个有意义的选择。

### 2.4 侦察与逃跑

- 点击任意节点时，均有一个确认/取消弹窗，其中战斗节点有一个额外的**侦查**选项。
- 战斗节点（小怪/精英）在路径地图上只显示节点类型，**不显示具体敌人情报**。
- 每个战斗节点**首次**侦查需要付出少量代价（如丢失少量水晶），下一次点开将免费查看已解锁的情报。
- 也就是说玩家可以付出大量代价，侦查接下来的每一个可选战斗分支，然后确定攻略其中一个；也可以直接闭眼选择进入。
- 一旦确认选择，则锁定路线（战斗失败出来也不能重新选了，方案既定）。

> 设计意图：侦察+逃跑机制继承了原版”走近boss才能看到信息”的不确定性，同时在爬塔框架下更加结构化。“侦察后逃跑有代价”保证了路径选择本身的重量感。

### 2.5 战斗系统

沿用已有的 FF14 boss战模拟器，包括：

- 职业技能循环（GCD / 能力技）
- boss / 精英怪技能表演与机制处理（AOE、分摊、击退等）
- 状态效果系统
- 简单的敌人 AI

小怪战斗相对简单快速（1–2 分钟），精英战斗有独特机制（2–3 分钟，不一定是纯战斗，也可能是怪异的机制马拉松），boss 战斗是完整的 FF14 式遭遇战（3–5 分钟）。

具体战斗细节参见模拟器文档（TODO: 链接）。

### 2.6 等级与水晶

- 击杀敌人获得**水晶**。
- 在**篝火**处消耗水晶进行升级，提升基础属性。
- 等级上限：**15 级**。
- 小怪战斗掉落少量水晶，精英战斗掉落较多水晶。

### 2.7 精英/Boss 掉落（3 选 1）

击杀精英怪或 boss 后必定掉落以下 3 件物品，玩家**选择其中 1 件**获取：

1. **1 把随机武器**
2. **1 颗随机魔晶石（A）**
3. **1 颗随机魔晶石（B）**

这意味着每次击杀精英/boss，玩家要在”切换职业/强化 build”之间做出抉择。

### 2.8 武器

| 属性        | 说明                                                     |
| ----------- | -------------------------------------------------------- |
| 职业绑定    | 切换武器 = 切换职业                                      |
| 固定词条 ×2 | 必定包含该职能的核心属性补正（如法杖 = 速度 + 魔法攻击） |
| 随机词条 ×1 | 从全局词条池中随机                                       |

武器决定当前职业，因此选择武器意味着可能需要重新调整魔晶石配装。

### 2.9 魔晶石

| 属性     | 说明                      |
| -------- | ------------------------- |
| 词条数量 | 随机 1–3 条               |
| 词条类型 | 随机（见词条体系）        |
| 词条等级 | 随机                      |
| 携带上限 | 无限制                    |
| 激活上限 | min(当前等级, 15)，最低 3 |

玩家可以自由选择激活哪些魔晶石。核心策略：在有限的激活槽位内，搭配出最优的词条组合。

### 2.10 词条体系（简化版）

采用简化的属性系统，分为以下大类：

| 词条类型 | 效果                 | 适用倾向      |
| -------- | -------------------- | ------------- |
| 物理攻击 | 提升物理技能伤害     | 近战 / 远物理 |
| 魔法攻击 | 提升魔法技能伤害     | 法系          |
| 防御     | 减少受到的伤害       | 通用 / 坦向   |
| 速度     | 缩短 GCD / 咏唱时间  | 通用          |
| 暴击率   | 提升暴击概率         | 通用          |
| 暴击伤害 | 提升暴击时的额外伤害 | 通用          |
| 生命值   | 提升最大 HP          | 通用 / 容错向 |

> 设计意图：词条类型足够少，让玩家能快速判断价值；但通过数值等级的随机性，仍然保持 build 多样性。不同职能有属性补正差异，但只要魔晶石搭配合理，任何职业都可以打出高伤害。

### 2.11 职业系统

### 初始职业（开局可选）

| 职业   | 职能     | 特色                 |
| ------ | -------- | -------------------- |
| 剑术师 | 物理近战 | 攻防均衡，容错率高   |
| 咒术师 | 魔法远程 | 爆发伤害高，需要读条 |
| 弓箭手 | 物理远程 | 持续输出，机动性强   |

### 解锁职业

通过精英/boss 掉落的武器解锁更多职业。每把武器对应一个特定职业，切换武器即切换职业。

| 解锁职业 | 职能     | 解锁方式     |
| -------- | -------- | ------------ |
| （TODO） | （TODO） | 拾取对应武器 |

### 2.12 篝火

篝火是路径上的安全节点，功能包括：

- **升级**：消耗水晶提升等级。
- **配装**：调整魔晶石激活组合、切换武器。
- **奉献**：献祭魔晶石以换取额外复活次数（用于 boss 战）。

### 2.13 Boss 战

每座塔的路径最终汇聚到一个 boss 节点：

- Boss 是一场完整的 FF14 式遭遇战，有多阶段机制。
- 使用此前收集的全部装备（武器 + 激活的魔晶石）进行攻略。
- **死亡机制**：boss 战中死亡可重新挑战，默认拥有 **3 次机会**。可通过篝火奉献魔晶石获取更多次数。
- 3 次机会耗尽则本局结束，仍然会结算奖励（但金币大幅减少）。

### 2.14 死亡惩罚

| 场景                | 惩罚                                         |
| ------------------- | -------------------------------------------- |
| 小怪/精英战斗中死亡 | 丢失少量水晶，从当前节点重试                 |
| Boss 战中死亡       | 消耗 1 次复活机会，重新挑战                  |
| 复活次数耗尽        | 本局结束，按当前收集情况结算（金币大幅减少） |

### 2.15 结算

| 结算因素       | 影响                         |
| -------------- | ---------------------------- |
| 魔晶石收集品质 | 收集到的魔晶石越好，金币越多 |
| Boss 击杀用时  | 用时越短，金币越多           |
| 是否通关       | 通关有大量额外金币           |

结算产出：**金币**（用于局外甲胄抽卡）。

## 3. 局外系统

### 3.1 甲胄（永久装备）

甲胄是局外获取的永久装备，跨局生效。

| 属性           | 说明                                          |
| -------------- | --------------------------------------------- |
| 部位           | 帽子 / 衣服 / 腰带 / 裤子 / 鞋（共 5 个部位） |
| 词条数量       | 随机 1–3 条                                   |
| 词条类型与等级 | 随机                                          |
| 穿戴规则       | 每个部位最多穿戴 1 件                         |
| 词条上限       | 5 部位 × 最多 3 词条 = 最多 15 条永久加成     |

### 3.2 甲胄商店（抽卡）

- 消耗**金币**进行抽卡，获得随机甲胄。
- 可使用**定轨券**保底：指定 3 个词条类型，但每个词条的数值（效果量）仍然随机。
- 可额外消耗定轨券，使其中 1 个词条**必定为最高稀有度**。

### 3.3 绝境战

独立于爬塔主玩法的高难度模式。

| 属性     | 说明                                   |
| -------- | -------------------------------------- |
| 形式     | 无存档点的高难度遭遇战                 |
| 职业选择 | 自由选择任意已解锁职业（武器加成固定） |
| 甲胄     | 可穿戴甲胄入场                         |
| 奖励     | 定轨券                                 |

定轨券的作用见 3.2 甲胄商店。

## 6. 设计备忘（讨论记录）

以下是设计过程中讨论过的关键决策与理由，供后续参考：

**为什么从大地图+毒圈改为爬塔？**
大地图+毒圈的实现成本高（大地图渲染、毒圈逻辑、移动系统），且与FF14的”站桩式”战斗体验存在节奏冲突。爬塔路径选择保留了”探索+决策”的核心，砍掉了最重的实现成本，同时天然支持断点续玩。

**为什么保留侦察+逃跑机制？**
继承原版”走近boss才能看到信息”的不确定性。在爬塔框架下，表现为”进入战斗节点后可以先看敌人信息再决定是否逃跑”。逃跑有少量代价（丢失水晶），保证路径选择的重量感。

**为什么 3 选 1 而不是分开掉落？**
强制取舍。每次击杀精英/boss 都要在”换职业可能性”和”强化当前 build”之间做决策，增加策略深度。

**为什么甲胄是抽卡而不是明码标价？**
配合定轨券形成”爬塔主线产出金币 → 绝境战产出定轨券 → 两者共同作用于甲胄获取”的双轨激励，让两种玩法都有存在意义。

**原版大地图设计的遗产**
原版的分阶段区域设计（外圈低级→内圈高级、debuff式毒圈限制发育时间）在爬塔版中自然映射为”越往上节点越难”的结构。毒圈的”限制发育时间”功能由有限的路径长度（节点数）替代。
